# Entity Documentation

## Conventions

- Validation rules live in DTOs, not entities. Entities only define column constraints (`nullable`, `unique`, `precision`, etc.).
- Relationships are set from the owning side (the entity that holds the FK). No unnecessary `@OneToMany` back-references unless the collection is actively read.
- `@ManyToOne` relationships use `FetchType.LAZY` to avoid N+1 queries.
- Indexes are added on columns that are frequently queried together.
- Mappers handle conversion between entities and DTOs.
- All timestamps use `OffsetDateTime` in UTC, set automatically via `@PrePersist` / `@PreUpdate`.
- All IDs are `UUID`, auto-generated by Hibernate.

## Entity relationship overview

```
User ──< Order ──< OrderItem >── Product
 │                                  │
 ├──< Rating >── Product            ├──< ProductImage >── FileEntity
 │                                  │
 ├──○ UserSettings                  ├──> Brand
 │                                  │
 └──>< Role                        └──> Category ──< Category (self-referencing tree)
```

`──<` one-to-many, `>──` many-to-one, `──><` many-to-many, `──○` one-to-one

## Entities

### User (`users`)

| Field         | Type             | Constraints              |
|---------------|------------------|--------------------------|
| id            | UUID             | PK, generated            |
| email         | String           | not null, unique         |
| name          | String           | not null                 |
| passwordHash  | String           |                          |
| provider      | AuthProvider     | not null, default LOCAL  |
| providerId    | String           |                          |
| emailVerified | boolean          | default false            |
| createdAt     | OffsetDateTime   | not null, immutable      |
| updatedAt     | OffsetDateTime   |                          |

**Relationships:**
- `roles` - `@ManyToMany` via `user_roles` join table. User is the owning side.
- `settings` - `@OneToOne` with `UserSettings`, cascaded. UserSettings shares the User's PK via `@MapsId`.

### Role (`roles`)

| Field | Type   | Constraints |
|-------|--------|-------------|
| id    | UUID   | PK, generated |
| name  | String | not null    |

No back-reference to User. Roles are queried through User, not the other way around.

### UserSettings (`user_settings`)

| Field             | Type    | Constraints       |
|-------------------|---------|-------------------|
| id                | UUID    | PK, shared with User via `@MapsId` |
| twoFactorEnabled  | boolean | default false     |

**Relationships:**
- `user` - `@OneToOne` with `@MapsId`. The settings row shares the same PK as the user.

### Product (`products`)

| Field       | Type             | Constraints              |
|-------------|------------------|--------------------------|
| id          | UUID             | PK, generated            |
| price       | BigDecimal(17,2) | not null                 |
| currency    | CurrencyProvider | not null, enum as STRING |
| name        | String           | not null                 |
| description | String           |                          |
| width       | BigDecimal       |                          |
| height      | BigDecimal       |                          |
| depth       | BigDecimal       |                          |
| weight      | BigDecimal       |                          |
| stock       | Integer          | not null                 |
| createdAt   | OffsetDateTime   | not null, immutable      |
| updatedAt   | OffsetDateTime   |                          |

**Relationships:**
- `brand` - `@ManyToOne(LAZY)`, optional. FK: `brand_id`.
- `category` - `@ManyToOne(LAZY)`, required. FK: `category_id`. Set via `product.setCategory(category)` (null-checked).
- `productImages` - `@OneToMany(mappedBy = "product")`. Actively used — `ProductMapper` reads the list, `addImage()` helper manages the bidirectional link.

### ProductImage (`product_images`)

| Field     | Type    | Constraints                                          |
|-----------|---------|------------------------------------------------------|
| id        | UUID    | PK, generated                                        |
| product   | Product | not null, FK: `product_id`                           |
| file      | FileEntity | not null, FK: `file_key`                          |
| sortOrder | Integer | not null, unique per product (`uk_product_sort`)     |

**Relationships:**
- `product` - `@ManyToOne(LAZY)`, required. Set via `Product.addImage()`.
- `file` - `@ManyToOne(LAZY)`, required. Points to a `FileEntity`.

### Brand (`brands`)

| Field | Type   | Constraints      |
|-------|--------|------------------|
| id    | UUID   | PK, generated    |
| name  | String | not null, unique |

Simple lookup entity. No back-references.

### Category (`categories`)

| Field    | Type     | Constraints              |
|----------|----------|--------------------------|
| id       | UUID     | PK, generated            |
| name     | String   | not null, unique         |
| isLeaf   | boolean  | not null, immutable      |

**Indexes:** `idx_categories_name` on `name`.

**Relationships:**
- `parentCategory` - `@ManyToOne(LAZY)`, optional. FK: `parent_category_id`. Self-referencing for tree structure.
- `subcategories` - `@OneToMany(mappedBy = "parentCategory")`, cascades PERSIST and MERGE, orphanRemoval. Actively used — `addChild()` enforces the leaf constraint and manages the bidirectional link. Saving a parent cascades to children.

Leaf categories hold products. Non-leaf categories hold subcategories. `addChild()` throws if called on a leaf.

### Order (`orders`)

| Field     | Type           | Constraints         |
|-----------|----------------|---------------------|
| id        | UUID           | PK, generated       |
| user      | User           | not null            |
| createdAt | OffsetDateTime | not null, immutable |
| updatedAt | OffsetDateTime |                     |

**Indexes:** `idx_orders_user_created` on `(user_id, created_at)`.

**Relationships:**
- `user` - `@ManyToOne(LAZY)`, required.

No back-reference to OrderItem. Order items are queried through `OrderItemRepository`.

### OrderItem (`order_items`)

| Field      | Type             | Constraints              |
|------------|------------------|--------------------------|
| id         | UUID             | PK, generated            |
| price      | BigDecimal(19,2) | not null                 |
| quantity   | Integer          | not null, CHECK >= 1     |
| totalPrice | BigDecimal(19,2) | computed on persist/update |
| createdAt  | OffsetDateTime   | not null, immutable      |
| updatedAt  | OffsetDateTime   |                          |

**Indexes:** `idx_order_items_order_added` on `(order_id, created_at)`.

**Relationships:**
- `order` - `@ManyToOne(LAZY)`, required.
- `product` - `@ManyToOne(LAZY)`, required.

`totalPrice` is automatically calculated as `price * quantity` in `@PrePersist` and `@PreUpdate`.

### Rating (`ratings`)

| Field     | Type           | Constraints                                     |
|-----------|----------------|-------------------------------------------------|
| id        | UUID           | PK, generated                                   |
| score     | int            | not null (1-5, validated in DTO)                |
| comment   | String         | max 1000 chars                                  |
| createdAt | OffsetDateTime | not null, immutable                             |
| updatedAt | OffsetDateTime |                                                 |

**Unique constraint:** `(user_id, product_id)` — one rating per user per product.

**Relationships:**
- `user` - `@ManyToOne(LAZY)`, required.
- `product` - `@ManyToOne(LAZY)`, required.

### FileEntity (`file_entities`)

| Field       | Type           | Constraints         |
|-------------|----------------|---------------------|
| key         | String(512)    | PK (S3 object key)  |
| url         | String         |                     |
| contentType | String         | not null            |
| sizeBytes   | Long           | not null            |
| createdAt   | OffsetDateTime | not null, immutable |

Standalone entity representing a file stored in S3. Referenced by `ProductImage` via `file_key` FK. No `updatedAt` — files are immutable once uploaded.

### Enums

- **AuthProvider** — `LOCAL`, plus any OAuth2 providers.
- **CurrencyProvider** — Currency codes (e.g. `EUR`). Stored as STRING.
